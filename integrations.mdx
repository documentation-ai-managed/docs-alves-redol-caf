## Setting Up Webhooks

Webhooks notify external services about events like doc updates or publishes. Follow these steps to configure one.

<Steps>
  <Step title="Generate Webhook URL" icon="link">
    In your dashboard, navigate to Settings > Webhooks. Click "New Webhook" and enter your endpoint URL, such as `https://your-webhook-url.com/alvesredol`.
  </Step>
  <Step title="Select Events" icon="bell">
    Choose events: `doc.updated`, `doc.published`, `comment.added`.
  </Step>
  <Step title="Test and Save" icon="play">
    Send a test event. Verify receipt before saving.
  </Step>
</Steps>

Handle incoming payloads with this example structure.

<CodeGroup tabs="Node.js,Python">
  ```javascript
  const express = require('express');
  const app = express();
  app.use(express.json());

  app.post('/alvesredol', (req, res) => {
    const event = req.headers['x-alvesredol-event'];
    const payload = req.body;
    console.log(`Event: ${event}`, payload);
    res.status(200).send('OK');
  });

  app.listen(3000);
  ```
  ```python
  from flask import Flask, request, jsonify

  app = Flask(__name__)

  @app.route('/alvesredol', methods=['POST'])
  def webhook():
      event = request.headers.get('X-AlvesRedol-Event')
      payload = request.json
      print(f"Event: {event}", payload)
      return jsonify({'status': 'OK'}), 200

  if __name__ == '__main__':
      app.run(port=3000)
  ```
</CodeGroup>

## API Access for Custom Connections

Build bespoke integrations using the REST API at `https://api.example.com/v1`.

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X POST https://api.example.com/v1/webhooks \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "url": "https://your-webhook-url.com/alvesredol",
      "events": ["doc.updated", "doc.published"]
    }'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/webhooks', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      url: 'https://your-webhook-url.com/alvesredol',
      events: ['doc.updated', 'doc.published']
    })
  });
  ```
</Request>

<Response tabs="201,400">
  ```json
  {
    "id": "wh_abc123",
    "url": "https://your-webhook-url.com/alvesredol",
    "events": ["doc.updated", "doc.published"],
    "active": true
  }
  ```
  ```json
  {
    "error": "Invalid URL",
    "message": "Webhook URL must be HTTPS"
  }
  ```
</Response>

<ParamField path="url" param-type="string" required="true">
  Your secure HTTPS endpoint.
</ParamField>

<ParamField body="events" param-type="array" required="true">
  Array of event names.
</ParamField>

<Expandable title="Advanced API Authentication" default-open="false">
  For production, use OAuth 2.0 flows. See the full API reference for scopes like `webhooks:write`.
</Expandable>
